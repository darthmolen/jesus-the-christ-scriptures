# Roadmap — Scriptures: Jesus The Christ

**Date:** 2026-06-07
**Status:** living roadmap (what's done, what's next)

## Done

- **Plan 01 — Content domain** (`JesusTheChrist.Core`): models, JSON loader, ScopeFilter, ContentService, gloss rule, reference-id scheme. Tested.
- **Plan 02 — Local data** (`JesusTheChrist.Data`): SQLite stores (read-marks, notes, settings), ProgressService, StreakService/Store. Tested.
- **Plan 03 / PR-A — MAUI scaffold** (merged, PR #10): `JesusTheChrist.App` (net10.0-android, MAUI + Material 3, MauiVersion 10.0.70), `JesusTheChrist.Presentation` lib, DI wiring, launches on device. Fixed a pre-existing Core build break under SDK 10.0.300.
- **Plan 03 / PR-B — Home / Challenge tracker** (open, PR #11): 53 sub-topics with progress rings under the "X / 2,196 references read" header; live to read-marks; tap → topic route. RingMath, LanguageResolver, HomeViewModel all TDD.

## Next (in priority order)

### Plan 04 — Readable reference feed + mark-read  ← BUILDING NOW (the "surprise")
The thing that makes it usable for study. Tap a sub-topic →
- vertical feed of **reference cards**: reference title on top, **target verse text** inline;
- **"Show context (±2)"** expander revealing surrounding verses;
- the **note gloss** shown only when it adds information (Reference.ShowGloss);
- a **read checkmark** per card that persists (ReadMarkStore) and makes the **Home rings advance**.
Replaces the PR-A `topic` stub page. ViewModels TDD; page verified on device.

### Plan 05 — Notes
Per-reference free-text note (NoteStore already exists): a note affordance on each card → editor sheet; card shows whether a note exists.

### Plan 06 — Settings & reading comfort
Settings page: font family/size (Slider), light/dark (Switch), language toggle, streak toggle, about/attribution screen. Persist via SettingsStore. Apply font/theme app-wide.

### Plan 07 — Secondary re-slice ("⇄ by book")
Inside a topic feed, regroup the same references by book; book+topic filter. (Topic-first stays primary.)

### Plan 08 — Bible-Only build flavor + branding
Second store flavor: `Scope.BibleOnly` constant via MSBuild config, app id/name/icon, two listings. ScopeFilter already supports it.

### Plan 09 — Polish & release prep
App icon/splash branding, accessibility pass, performance pass on the long feed, attribution screen finalize. Then the licensing gate (Intellectual Reserve permission) before any public release — see design-spec §9.
